# 微信消息整理功能使用指南

> 把微信群聊天记录粘进来，AI 自动识别出排练 / 路演的日期、地点、服装、谁来谁不来，进入「待确认」队列，你点确认才入库。

---

## 一、它能做什么、不能做什么

**能做** ✅
- 识别**排练**：日期、时间、地点、服装、备注
- 识别**路演**：活动名、城市、日期、时间、地点
- 识别**出席态度**：谁说"我去"、谁说"我不去"、谁说"我看看"
- 处理相对日期：把"明天 7 点"、"周六"、"这周五"换算成绝对日期
- 处理改口：同一件事先约 7 点后改 8 点 → 只取最终版

**不能做** ❌
- 自动同步聊天（每次都需要手动粘贴一段进来）
- 给非车主用：只有歌曲车主能粘贴和确认（因为只有车主能建排练/路演）
- 识别拍摄计划 / 装搭照片等非排练非路演的信息（之后再加）

## 二、首次配置：拿 DeepSeek 的 API key（5 分钟）

DeepSeek 是国内最便宜也最稳的大模型，按用量付费，整理一段聊天大约花 **0.001 ~ 0.005 元**（不到一分钱）。

1. 浏览器打开 https://platform.deepseek.com
2. 用微信登录（手机扫码即可）
3. 左侧「**API Keys**」→「**创建 API key**」
4. 给 key 起个名字（比如 `kpop-server`）→ 复制下来（只能复制一次，关掉就看不到）
5. 顺便去「**充值**」→ 充 **10 元**（够你整理几千次聊天）

把 key 配到服务器：

```bash
# SSH 登录服务器
ssh lynn@47.93.9.199

# 改 .env，把 DEEPSEEK_API_KEY 填上
nano ~/kpop-server/.env
# 找到这行：
# DEEPSEEK_API_KEY=
# 改成：
# DEEPSEEK_API_KEY=sk-你刚复制的那一长串
# Ctrl+O 保存，Ctrl+X 退出

# 重启服务让它读到新配置
pm2 restart kpop-server
pm2 logs kpop-server --lines 5
```

看到 `🚀 K-pop server v2 running` 就 OK 了。

## 三、怎么用

### 第 1 步：复制聊天记录

在微信群里：
1. **长按**任意一条消息
2. 选「**多选**」
3. 勾选你想整理的消息（一般是最近几条排练 / 路演相关的）
4. 右下角「**合并转发**」→ 转发给「**文件传输助手**」（或自己另一个号）
5. 打开文件传输助手 → 长按那条转发的消息 → 选「**复制**」

> 💡 也可以直接长按单条消息选「复制」，再多次粘贴拼起来。看你方便。

### 第 2 步：粘贴 + 整理

1. 浏览器打开 http://47.93.9.199:8080，登录
2. 点顶部「**📥 整理消息**」tab
3. 把聊天记录粘进大文本框
4. （可选）下拉框选「关联到哪首歌」—— 选了能让 AI 更准；不选 AI 自己判断
5. 点「**✨ 帮我整理**」
6. 等 5-10 秒，AI 思考中

### 第 3 步：审核

下方「待确认」列表会出现解析结果卡片：

- **关联歌曲下拉**：AI 如果没猜中，可以手动改
- **字段输入框**：日期、时间、地点、服装、备注 —— 都可以直接改
- **出席预填**：每个人的小标签可以**点击**循环切换：`✓ 来` → `? 看看` → `✗ 不来`
- **🤖 AI 备注**：AI 自己说的解析依据 / 不确定之处

确认无误 → 点「**✅ 确认添加**」→ 写入对应歌曲的排练 / 路演表
不要这条 → 点「**✗ 不要这条**」→ 从队列移除

## 四、常见情况

### Q1：AI 把日期识别错了
直接改卡片上的日期输入框，点确认即可。
如果错得很多，可以在聊天记录粘贴前手动加一行 `今天是 2026-05-12`，AI 会以这个为基准。

### Q2：AI 没识别到我朋友的出席
出席只会标到 AI **明确识别出**态度的人；模糊的（比如沉默 / 表情包）AI 会忽略。
你可以在卡片上手动点 chip 切换状态，或者确认后去歌曲详情里改。

### Q3：识别出来的人不在这首歌的队伍里
出席记录会被自动忽略（数据库约束：只能给歌曲成员标出席）。
解决：先去歌曲详情「队伍」页加好友，再回来确认。

### Q4：群里有些路人发言不是我朋友
AI 只会识别你"成员列表"里的人（=你 + 你的好友），路人会被忽略。

### Q5：花了多少钱怎么看
登录 https://platform.deepseek.com → 左侧「用量」可以查实时消耗。

### Q6：能换其他 AI 服务吗
可以。比如换通义千问：`.env` 里改成
```
DEEPSEEK_API_KEY=你的通义key
AI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
AI_MODEL=qwen-plus
```
重启 `pm2 restart kpop-server` 即可。Kimi、智谱 GLM 同样兼容 OpenAI 协议，照着改。

## 五、技术原理（给好奇心强的人）

1. 你粘贴的文本 + 你的歌曲列表 + 好友列表 + 今天日期 → 拼成 prompt
2. 后端调 DeepSeek API（OpenAI 兼容协议），让它输出严格 JSON
3. 后端校验 JSON、过滤无效字段（song_id、user_id 必须在白名单里）
4. 写入 `pending_items` 表，等待你点确认
5. 你确认 → 把数据写进 `rehearsals` / `performances` 表，标记 pending_items 为 confirmed

整个过程零暴露给 AI 服务的信息只有：你这次粘的文本 + 你的歌名 + 你好友的昵称。
**密码、邮箱、邀请码、其他歌的数据都不会被发出去。**

---

## 六、限流说明

防止误操作把钱烧光：
- 每个用户每分钟最多调用 **6 次**「✨ 帮我整理」
- 单次粘贴文本上限 **5 万字**（够整理几十条群聊了）

如果想调整，改 `server.js` 里 `parseLimiter` 的 `max` 值。
