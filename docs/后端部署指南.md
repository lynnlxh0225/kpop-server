# 后端部署指南 (v1.0)

> 把刚写好的 `kpop-server/` 文件夹部署到你的阿里云服务器，让朋友们能通过云端注册登录、协作管理。
>
> ⏱ 预计用时 30-45 分钟（含网络下载时间）

---

## 流程总览

```
本机 Mac                                阿里云服务器
┌──────────────┐                       ┌─────────────────────┐
│ kpop-server/ │  ─── scp 上传 ──→     │ ~/kpop-server/      │
└──────────────┘                       │  npm install        │
                                       │  配置 .env          │
                                       │  pm2 start          │
                                       │  ↓                  │
                                       │ Node 后端 :3000     │
                                       │  ↓                  │
                                       │ Nginx 代理 /api/    │
                                       │ + 静态文件 /public/ │
                                       └─────────────────────┘
                                                ↓
                                         浏览器访问 :8080
```

## 第 1 步：本机准备 kpop-server 文件夹

打开 Mac 本地终端（**不是 SSH 那个**），执行：

```bash
# 把 kpop-server 文件夹从 outputs 复制到桌面
cp -R "/Users/didi/Library/Application Support/Claude/local-agent-mode-sessions/eee5bdf3-af79-4797-92f7-2a46ee362e5a/90dbe772-2687-42da-95c2-4f311956d6d4/local_a1aaf578-c8db-4626-aa9f-a8c3128ad89d/outputs/kpop-server" ~/Desktop/

# 验证文件齐全
ls -la ~/Desktop/kpop-server/
```

应该看到：

```
package.json
server.js
.env.example
.gitignore
ecosystem.config.cjs
public/
```

## 第 2 步：上传整个文件夹到服务器

**本机终端**继续：

```bash
scp -r ~/Desktop/kpop-server lynn@47.93.9.199:~/
```

输入 lynn 密码后开始上传。看到每个文件 `100%` 就是成功了。

## 第 3 步：服务器上安装依赖

SSH 登录服务器：

```bash
ssh lynn@47.93.9.199
```

登录后：

```bash
cd ~/kpop-server

# 安装依赖（会下载 6 个包，约 15-30 秒）
npm install

# 创建日志目录
mkdir -p logs
```

⚠️ 如果 `npm install` 报错，告诉我错误信息。

---

## 第 4 步：配置 .env（关键！）

```bash
# 复制配置模板
cp .env.example .env

# 生成一个强 JWT 密钥
openssl rand -base64 48
```

会输出一串类似这样的随机字符：
```
7Vk3pL8qXm4nN2cZ9bA5jK6sF3wQ1tH7gY8vR4eU2iO5xS3aD1mB9oP6lQ8r
```

**复制这串字符**，然后：

```bash
nano .env
```

进入编辑界面后：
1. 找到 `JWT_SECRET=...` 那一行
2. 把等号后面的内容**整行替换**成刚才复制的随机字符
3. 按 `Ctrl + O` → 回车（保存）
4. 按 `Ctrl + X`（退出）

最终 `.env` 文件应该类似：

```
PORT=3000
JWT_SECRET=7Vk3pL8qXm4nN2cZ9bA5jK6sF3wQ1tH7gY8vR4eU2iO5xS3aD1mB9oP6lQ8r
DB_PATH=./data/data.db
REQUIRE_INVITE=yes
CORS_ORIGIN=*
```

> 💡 `REQUIRE_INVITE=yes` 表示：除了第一个注册的用户外，其他人都必须用邀请码注册。这样可以防止陌生人乱注册占用你的服务器资源。

---

## 第 5 步：用 PM2 启动后端

```bash
pm2 start ecosystem.config.cjs
pm2 save                # 保存当前进程列表
pm2 startup             # 设置开机自启
```

`pm2 startup` 会打印一行 `sudo env PATH=... pm2 startup ...` 的命令，**把它整行复制粘贴**回终端执行。

验证服务跑起来了：

```bash
pm2 list
```

应该看到 `kpop-server` 状态 `online`、绿色。

测试 API：

```bash
curl http://127.0.0.1:3000/api/health
```

应该返回 `{"status":"ok","time":"..."}`。

---

## 第 6 步：修改 nginx 配置加上 /api/ 代理

```bash
sudo tee /etc/nginx/sites-available/kpop > /dev/null << 'EOF'
server {
    listen 8080;
    server_name _;
    root /home/lynn/kpop-server/public;
    index index.html;

    # 前端静态文件
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 转发
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
EOF

# 测试配置 + 重载
sudo nginx -t
sudo systemctl reload nginx
```

> 注意：上面命令把 root 改成了 `/home/lynn/kpop-server/public`。旧的 `/var/www/kpop/index.html` 不会再用，但你可以保留它当备份。

## 第 7 步：浏览器测试

打开浏览器访问 `http://47.93.9.199:8080`，应该看到**登录页**（粉紫色渐变背景）。

### 注册第一个用户（你自己）

1. 点「立即注册」
2. 填昵称、邮箱、密码（≥6位）、emoji 头像
3. **邀请码留空**（你是第一个用户，可以跳过）
4. 点「注册」

注册成功后会自动登录、进入主页，看到「Lynn的舞团」（用你昵称命名）已经创建好了。

### 邀请朋友

1. 点右上角的「⚙️ 设置」
2. 在「当前小组」卡片里复制 8 位邀请码（如 `AB12CD34`）
3. 或者复制邀请链接 `http://47.93.9.199:8080/?invite=AB12CD34`
4. 把链接发给朋友，她们点开就会跳到注册页，邀请码自动填好

朋友注册成功后会自动加入你的小组，能看到你创建的所有歌曲/排练/路演。

## 故障排查

### 浏览器访问 `:8080` 显示 502 Bad Gateway
说明 nginx 能连上但后端没起来。在服务器执行：
```bash
pm2 list
pm2 logs kpop-server --lines 50
```
看后端是否 crash。常见原因：`.env` 配置错、端口被占用。

### 注册时点了没反应
打开浏览器开发者工具（`F12` 或 `Cmd+Option+I`），切到「Network」标签，再点注册，看 `/api/auth/register` 请求的响应内容。如果是 4xx/5xx，把错误粘给我。

### "404 API 不存在"
说明 nginx 没把 `/api/` 转给后端。检查 nginx 配置文件、重新执行第 6 步的 `tee` 命令。

### 想清空数据重来
```bash
pm2 stop kpop-server
rm ~/kpop-server/data/data.db
pm2 start kpop-server
```
重启后数据库会自动重建。

### 后端代码升级（以后我帮你改了功能）
```bash
# 在本机
scp -r ~/Desktop/kpop-server-v2/* lynn@47.93.9.199:~/kpop-server/

# 在服务器
cd ~/kpop-server
npm install   # 如果改了依赖
pm2 restart kpop-server
```

## 自动备份数据库（推荐设置）

```bash
mkdir -p ~/backups

cat > ~/backup.sh << 'EOF'
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
cp ~/kpop-server/data/data.db ~/backups/data_$DATE.db
find ~/backups -name "data_*.db" -mtime +30 -delete
EOF

chmod +x ~/backup.sh

# 添加到 cron：每天凌晨 3 点
(crontab -l 2>/dev/null; echo "0 3 * * * /home/lynn/backup.sh") | crontab -
```
