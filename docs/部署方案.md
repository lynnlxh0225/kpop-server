## 一、整体架构（一图看懂）

```
                            互联网
                              │
                              ▼
            ┌─────────────────────────────────┐
            │   special-lifejourney.com (域名)│
            │        ↓ DNS 解析到             │

            └─────────────────────────────────┘
                              │

            ┌─────────────────────────────────┐
            │     Nginx (80/443端口)          │  ← 反向代理 + HTTPS
            │   - 静态文件直接返回 HTML/CSS/JS   │
            │   - /api/* 转发给后端 Node 服务   │
            └─────────────────────────────────┘
                       │              │
                  静态文件         /api/*
                       │              │
                       ▼              ▼
            ┌─────────────┐  ┌──────────────────┐
            │ public/     │  │  Node.js 后端    │ ← PM2 守护进程
            │ index.html  │  │  Express + JWT   │
            │ 前端 SPA    │  │  ↓                │
            └─────────────┘  │  better-sqlite3   │
                             │  ↓                │
                             │  data.db (单文件) │
                             └──────────────────┘
                                      │
                                      ▼
                             每天 03:00 自动备份
                             /backups/ + 邮箱
```

**关键设计取舍（给你的情况量身定做）：**

| 项目 | 选择 | 为什么 |
|------|------|--------|
| 后端语言 | **Node.js** | 与前端同一种语言，社区资源最多，AI助手最好用 |
| Web框架 | **Express** | 最简单、最稳定，全网教程最多 |
| 数据库 | **SQLite（单文件）** | 不用单独装数据库服务、不占内存、单文件好备份 |
| 进程守护 | **PM2** | 一行命令启动、崩溃自动重启 |
| 反向代理 | **Nginx** | 标配 |
| HTTPS | **Let's Encrypt 免费证书** | 一行命令自动续期 |
| 部署方式 | **Git pull + PM2 reload** | 改完代码 push 到 GitHub，登录服务器 pull 即可更新 |

## 三、立刻就能做的：服务器初始化（备案期间）

### 3.1 SSH 登录服务器

打开 macOS 自带的 **终端**（Terminal）应用：

```bash
ssh root@47.93.9.199
```

第一次会问 yes/no，输入 `yes`，然后输入密码（阿里云控制台「重置密码」可以设置）。

> 💡 如果想免密码登录，参考阿里云文档配置 SSH 密钥对。

### 3.2 系统更新 + 基础工具

登录服务器后，**逐行复制粘贴**以下命令：

```bash
# 更新系统
apt update && apt upgrade -y

# 安装常用工具
apt install -y curl wget git vim ufw

# 设置时区到北京
timedatectl set-timezone Asia/Shanghai

# 设置防火墙：只开 22（SSH）、80（HTTP）、443（HTTPS）、8080（临时测试）
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 8080/tcp
ufw --force enable
ufw status
```

### 3.3 创建普通用户（更安全，以后用这个用户操作）

```bash
# 创建一个叫 lynn 的用户（你也可以改名）
adduser lynn
# 跟着提示设置密码，其他直接回车跳过

# 给 lynn 加 sudo 权限
usermod -aG sudo lynn
```

之后退出 root，用 `ssh lynn@47.93.9.199` 重新登录。

### 3.4 安装 Node.js（用 nvm，最佳实践）

切换到 `lynn` 用户后：

```bash
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc

# 安装最新 LTS 版本的 Node
nvm install --lts
node -v   # 应该输出 v20.x.x
npm -v
```

### 3.5 安装 Nginx 和 PM2

```bash
# Nginx
sudo apt install -y nginx

# 启动 + 设置开机自启
sudo systemctl start nginx
sudo systemctl enable nginx

# PM2 全局安装（管理 Node 进程的工具）
npm install -g pm2
```

完成这一步后，浏览器访问 `http://47.93.9.199` 会看到 nginx 欢迎页 → 说明服务器基础环境搞定。

### 3.6 把现有 HTML 原型先丢上去（临时跑起来）

```bash
# 创建一个目录存放网站
sudo mkdir -p /var/www/kpop
sudo chown -R lynn:lynn /var/www/kpop

# 把 HTML 上传到这个目录（具体方法见下方两种之一）
```

**上传方式 A（最简单）：scp 命令**

在你的 **本机终端**（不是服务器上）执行：

```bash
scp ~/Downloads/kpop-dance-manager.html lynn@47.93.9.199:/var/www/kpop/index.html
```

**上传方式 B：通过 Git**（推荐，将来用这个）

后面会详细说。

**配置 nginx 让它能访问到：**

```bash
sudo vim /etc/nginx/sites-available/kpop
```

按 `i` 进入编辑模式，粘贴：

```nginx
server {
    listen 8080;
    server_name _;
    root /var/www/kpop;
    index index.html;
    location / {
        try_files $uri $uri/ =404;
    }
}
```

按 `Esc`，输入 `:wq` 保存退出。

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/kpop /etc/nginx/sites-enabled/
sudo nginx -t        # 测试配置无误
sudo systemctl reload nginx
```

✅ 现在浏览器访问 `http://47.93.9.199:8080` 应该能看到 K-pop 管理器了！把这个临时地址发给朋友先用着。

> ⚠️ 备案没下来之前**禁止**用 80 端口绑域名，否则可能被阿里云拦截。

## 四、备案完成后：切换到域名 + HTTPS

### 4.1 域名解析（在阿里云控制台）

进入「域名控制台」→ 选 `special-lifejourney.com` →「DNS解析」→ 添加记录：

| 记录类型 | 主机记录 | 解析线路 | 记录值 |
|---------|---------|---------|--------|
| A | @ | 默认 | 47.93.9.199 |
| A | www | 默认 | 47.93.9.199 |

等 5-10 分钟生效。

### 4.2 把 nginx 配置改成 80 端口 + 域名

```bash
sudo vim /etc/nginx/sites-available/kpop
```

改成：

```nginx
server {
    listen 80;
    server_name special-lifejourney.com www.special-lifejourney.com;
    root /var/www/kpop;
    index index.html;

    # 静态前端文件
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 转发
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

```bash
sudo nginx -t && sudo systemctl reload nginx
```

### 4.3 申请 HTTPS 证书（免费、自动续期）

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d special-lifejourney.com -d www.special-lifejourney.com
```

跟着提示填邮箱、同意条款，选「2」自动跳转HTTPS。完成后 nginx 配置会自动改成 HTTPS。

证书 90 天到期，但 certbot 已经自动设置了 cron 任务定期续期，你不用管。

✅ 现在访问 `https://special-lifejourney.com` 应该能看到网站，带小绿锁。

### 5.2 数据库表结构（SQLite）

```sql
-- 用户
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  email TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  name TEXT NOT NULL,
  avatar TEXT DEFAULT '👤',
  created_at INTEGER NOT NULL
);

-- 小组（一个舞团）
CREATE TABLE groups (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  owner_id INTEGER NOT NULL,
  invite_code TEXT UNIQUE NOT NULL,
  created_at INTEGER NOT NULL
);

-- 小组成员（多对多）
CREATE TABLE group_members (
  group_id INTEGER,
  user_id INTEGER,
  role TEXT DEFAULT 'member',  -- 'owner' | 'admin' | 'member'
  joined_at INTEGER,
  PRIMARY KEY (group_id, user_id)
);

-- 歌曲
CREATE TABLE songs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  group_id INTEGER NOT NULL,
  title TEXT NOT NULL,
  artist TEXT,
  type TEXT DEFAULT 'new',     -- 'new' | 'classic'
  notes TEXT,
  created_at INTEGER NOT NULL
);

-- 歌曲队伍位置（哪个成员翻跳哪个位置）
CREATE TABLE song_members (
  song_id INTEGER,
  user_id INTEGER,
  position TEXT,
  PRIMARY KEY (song_id, user_id)
);

-- 排练
CREATE TABLE rehearsals (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  song_id INTEGER NOT NULL,
  date TEXT NOT NULL,
  time TEXT,
  location TEXT,
  outfit TEXT,
  notes TEXT,
  created_at INTEGER NOT NULL
);

-- 排练出席
CREATE TABLE rehearsal_attendance (
  rehearsal_id INTEGER,
  user_id INTEGER,
  status TEXT,                 -- 'yes' | 'maybe' | 'no'
  PRIMARY KEY (rehearsal_id, user_id)
);

-- 路演
CREATE TABLE performances (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  song_id INTEGER NOT NULL,
  name TEXT NOT NULL,
  city TEXT,
  date TEXT NOT NULL,
  time TEXT,
  location TEXT,
  outfit TEXT,
  status TEXT DEFAULT 'planned', -- planned|confirmed|done|canceled
  notes TEXT,
  created_at INTEGER NOT NULL
);

-- 路演出席
CREATE TABLE performance_attendance (
  performance_id INTEGER,
  user_id INTEGER,
  status TEXT,
  PRIMARY KEY (performance_id, user_id)
);
```

### 5.4 服务器上首次部署后端的命令

> 假设代码已经推到 GitHub `github.com/你的用户名/kpop-server`

## 六、数据备份与运维

### 6.1 每天自动备份数据库

```bash
# 创建备份目录
mkdir -p ~/backups

# 创建备份脚本
vim ~/backup.sh
```

粘贴以下内容：

```bash
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
cp ~/kpop-server/data/data.db ~/backups/data_$DATE.db
# 只保留最近 30 天的备份
find ~/backups -name "data_*.db" -mtime +30 -delete
```

保存后：

```bash
chmod +x ~/backup.sh

# 添加到 crontab：每天凌晨3点自动备份
crontab -e
# 在末尾加这一行：
0 3 * * * /home/lynn/backup.sh
```

### 6.2 把备份发到自己邮箱（防止服务器爆了）

进阶方案：用 [rclone](https://rclone.org) 同步到云盘（Google Drive、阿里云盘等）。这个等你跑稳了再说。
