# 手机日历订阅指南

> 把网站里的排练/路演自动同步到 iPhone / Android 系统日历，前一天自动提醒，不用打开网站就能查。

---

## 一、它能做什么

订阅之后，下面这些事件会自动出现在你的系统日历里：
- 你**车主**的歌的所有排练（含未来 + 过去）
- 你**加入的歌**的所有排练（你出席不是「不来」的）
- 所有路演（你出席不是「不参」、状态不是「未演出」的）

每个事件里能看到：歌名、车主、地点、服装/装搭、你的出席状态、备注。

**特性**：
- ✅ 系统级通知：iOS 默认会在事件前一天和当天提前提醒
- ✅ 单向同步：网站改了，日历自动跟上（iOS 一般 1 小时刷一次，最快 5 分钟）
- ✅ 跨设备：iPhone / Mac / Android / Google Calendar 都通
- ❌ 不能反向：在日历里改时间，网站不知道。改还是要回网站改。

## 二、拿到你的订阅链接

1. 浏览器打开 K-pop 管理器，登录
2. 点顶部「⚙️ 设置」tab
3. 找到「📆 排练日历订阅」卡片
4. 看到一段长链接，形如 `https://special-lifejourney.com/api/cal/AbCxxxx.ics`
5. 点「复制链接」按钮

> ⚠️ 这个链接相当于"只读你日程"的钥匙。**不要发到群里、不要截图给朋友看**。被泄漏立即点「🔄 重置订阅链接」生成新的。

## 三、iPhone 怎么添加

### 方法 A：在网站里点「📱 一键添加到 iPhone」按钮（最简单）

在 iPhone 的 Safari 里打开 K-pop 管理器 → 设置页 → 点紫色那个「📱 一键添加到 iPhone」按钮 → Safari 会弹出「打开『日历』？」→ 选「打开」→ 在弹出的对话框点「订阅」→ 完事。

### 方法 B：手动添加（如果方法 A 失败）

1. 先在网站「复制链接」
2. 打开 iPhone「设置」app
3. 找到「日历」→「账户」→「添加账户」→「其他」
4. 选「添加已订阅的日历」
5. 把链接粘到「服务器」框里，把开头的 `https://` 改成 `webcal://`（即整个链接像 `webcal://special-lifejourney.com/api/cal/AbCxxxx.ics`）
6. 点「下一步」→「存储」

打开 iPhone 自带「日历」app，应该看到一个新的日历叫「你的名字 的 K-pop 日程」。

### 调整刷新间隔（可选）

iOS 默认大概 1 小时刷新一次，想更频繁：
- 设置 → 日历 → 账户 → 选你刚添加的订阅 → 找「刷新」选项 → 改为「每 15 分钟」

## 四、Android 怎么添加

Android 系统日历不直接支持订阅，需要走 **Google Calendar** 中转（多数 Android 自带）：

1. 用电脑或手机浏览器打开 https://calendar.google.com
2. 左侧「其他日历」旁边 → 点「+」→ 选「通过网址添加」
3. 把你的订阅链接（开头是 https:// 那个）粘进去 → 点「添加日历」
4. 等 1-2 分钟，Google Calendar 拉一次
5. 手机上的 Google Calendar app 自动同步

> 📱 备注：如果你的 Android 没有 Google 服务（比如华为/小米国行），可以装第三方日历 app（如「日历 ICS Importer」、「ICSx⁵」），原理一样。

---

## 五、Mac 怎么添加

更简单——任何 Mac 上的浏览器里点「📱 一键添加到 iPhone」按钮，会自动弹 Mac 日历 app（不止 iPhone 能用）。或者：

1. Mac 日历 app → 文件 → 新建日历订阅
2. 粘订阅链接（https:// 开头那个）→ 订阅
3. 自动同步到所有登录同一 iCloud 的设备

## 六、常见情况

### Q1：订阅后没显示任何事件？
- 你必须**至少有一首歌、那首歌至少有一个排练或路演**，订阅才会有内容
- 你出席状态全设成「不来 / 不参」也会导致空（这是按设计）

### Q2：刚在网站添加的排练，日历里没立刻出现？
- iOS 拉取有延迟，**5 分钟到 1 小时**。等等或者手动下拉「日历」app 刷新

### Q3：我重置 token 之后，朋友们订阅的会怎样？
- 朋友是订阅他们**自己的链接**，不是你的。你重置只影响你自己。
- 如果你不小心把链接分享给了别人，他们订阅了你的日历——重置立即让他们的链接失效

### Q4：删除订阅怎么操作？
- iPhone：设置 → 日历 → 账户 → 选订阅 → 删除账户
- Google Calendar：网页端 → 其他日历 → 找到 → ⋮ → 取消订阅

### Q5：删了想重新加？
- 重新走第三节的流程即可。链接和原来的一样（除非你点过"重置 token"）

### Q6：能不能让朋友们一键订阅同一个日历？
- 不行也不应该。每个人订阅自己的，看到的是 ta 自己的出席安排。强行让 N 个人订阅同一个会有隐私问题。

## 七、技术原理

后端生成标准 [iCalendar (RFC 5545)](https://datatracker.ietf.org/doc/html/rfc5545) 格式的 .ics 文本，每个排练/路演是一个 VEVENT，包含时间、地点、描述。

订阅链接里的 token 是 32 字节随机串（base64url 编码），与你的账号绑定。只能用于**读取**你的日程，不能登录、不能改数据。重置 token 后旧链接立即返回 404。

代码在 `server.js` 的 `buildIcsFor()` 函数，路由 `GET /api/cal/:token.ics` 公开（不需要登录），靠 token 做认证。
